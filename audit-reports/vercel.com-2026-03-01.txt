8. TOP 3 CRITICAL ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ① No /agents.md (8 pts left on the table)
     Vercel already ships /llms.txt — the institutional awareness is there.
     /agents.md is the natural companion: while llms.txt says "here is our
     content," agents.md says "here is how to interact with us." For a
     company actively building AI infrastructure (AI Gateway, Vercel Agent,
     Fluid Compute), shipping an agents.md is a credibility signal to the
     agent operator community. It is a public/ directory commit away.

   ② h1 → h3 skip followed by h2 (visual-first heading structure)
     The hero uses h3 for the tab-section labels ("Your product, delivered.",
     "Agents", "AI Apps", etc.) while subsequent page sections use h2. This
     means the semantic outline reads as: one h1 → six h3s → eight h2s.
     An agent using heading traversal to summarise the page will treat the
     h3 content (tab labels) as grandchild nodes and the h2 content (main
     sections) as child nodes — the opposite of their visual importance.
     Fix: promote the section-label h3 to h2, and use a styled <p> or
     <span> for the decorative tab-group title.

   ③ 5 unlabeled integration/partner logo links in the hero
     @e18–@e22 are anchor elements wrapping images in the hero section with
     no alt text or aria-label. An agent trying to understand "what
     integrations does Vercel support?" by reading links will find these
     blank. Adding aria-label="[Provider] integration" or alt text to the
     wrapped images takes minutes and closes the gap.

   QUICK WINS  (implement in < 1 week)
   ──────────────────────────────────────────────────────────────────

   • Add /agents.md to public/
     Content should include: permitted agent interactions (API discovery,
     deployment status checks, template browsing), rate limit pointers,
     preferred entry points for agent use cases, and a link to the
     Vercel Agent product for agent operators. Given Vercel's position in
     the AI infra market, this file would likely become a reference
     implementation for the ecosystem. +8 pts; Grade B → A.

   • Add aria-label to the 5 hero logo/integration links (e18–e22)
     Identify the companies/frameworks displayed and add aria-label or
     descriptive alt text to the image inside each anchor. One PR, zero
     design impact. +partial ARIA score.

   • Explicitly allow AI bots in robots.txt
     Adding named entries (GPTBot, ClaudeBot, CCBot, anthropic-ai) with
     `Allow: /` is a low-cost, high-signal change. Moves robots.txt score
     from 2/3 to 3/3.

   STRATEGIC  (architectural / design system level)
   ──────────────────────────────────────────────────────────────────

   • Fix the heading hierarchy in the hero tab-group section
     Promote the tab-group label to h2 and use h3 for the individual tab
     panel headings. This aligns the semantic outline with the visual
     hierarchy (hero section is the most prominent content, not a
     grandchild node). Requires coordinated change across the marketing
     site design system.

   • Add a <footer> landmark (role="contentinfo")
     The "Vercel Directory" navigation is semantically the footer but is
     wrapped in a <nav> rather than a <footer>. Best practice is a <footer>
     or <div role="contentinfo"> wrapping the entire footer area, with the
     nav inside it. This correctly separates navigation from contentinfo.

   • Add a WebSite schema with SearchAction
     The Organization schema is excellent. Adding a WebSite schema with
     a potentialAction of SearchAction (pointing to vercel.com/search or
     docs search) enables agents and Google to surface Vercel's sitelinks
     search box and gives LLMs a structured "how to search this site" signal.
