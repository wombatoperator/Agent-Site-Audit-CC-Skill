1. EXECUTIVE SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   Grade : F    Score : 11 / 100

   advertising.roku.com does not resolve to a Roku-owned or Roku-operated
   advertising platform. Every connection attempt during this audit routed
   through a shared CDN or DNS infrastructure that served rotating content
   from unrelated adtech companies — including Criteo, Index Exchange,
   Magnite, Lotame, VideoAmp, and PubMatic — in non-deterministic sequence.
   The agent-browser session opened the target URL successfully (initial
   page title: "It's all on Roku | Roku Advertising") but immediately
   redirected to a third-party domain on every subsequent load, landing on
   a different company's homepage each time.

   This is not an accessibility problem — it is a DNS or domain-ownership
   infrastructure failure. No Roku advertising content is accessible at
   this URL. An AI agent attempting to learn about, engage with, or
   transact through Roku's advertising platform via this URL would be
   misled to unrelated competitors' sites on every session.

   The F grade reflects that the domain fails at the most fundamental
   level: it cannot deliver its own content to a browser agent. All scoring
   dimensions that require stable content receive a zero or near-zero by
   necessity. The 11 points awarded reflect the observation that the
   rotating sites themselves (Criteo, Index Exchange) deliver well-formed
   HTML with reasonable accessibility trees — but those belong to other
   companies and cannot be credited to advertising.roku.com.

8. TOP 3 CRITICAL ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ① DNS / Domain Infrastructure Failure (catastrophic — all scores zeroed)
     advertising.roku.com does not serve Roku content. Every session
     load redirected to a different adtech company's homepage via what
     appears to be a shared CDN or misconfigured DNS record. The rotating
     destinations included Criteo, Index Exchange, Magnite, Lotame,
     VideoAmp, and PubMatic — all competitors or adjacent adtech players.
     This is the most severe possible finding: the domain exists but
     delivers entirely wrong content. An AI agent sent to this URL to
     research or transact with Roku advertising would be fed false
     structured data, competitor content, and mismatched brand signals.
     Root cause is likely a lapsed DNS record pointing to a shared IP
     block now serving other tenants, or a CDN misconfiguration. This
     must be resolved before any other audit dimension is relevant.

   ② Zero Roku-Controlled Content (all structured data invalid)
     Because no Roku content loads, there is no JSON-LD, no valid OG
     tags, no Roku-authored meta description, and no llms.txt that an
     LLM or AI agent can use. The structured data found belongs to 4+
     different companies across sessions. This creates a disinformation
     risk: any LLM that crawls this URL and caches the response will
     associate advertising.roku.com with Criteo, Index Exchange, or
     another vendor depending on which session was cached.

   ③ Navigation is Non-Functional for Agents
     The redirect chain causes ERR_ABORTED on direct navigation attempts,
     prevents back() from returning to any Roku state, and routes clicks
     on navigation elements to unrelated domains (e.g., clicking "Contact
     Sales" on a Criteo nav resulted in landing at indexexchange.com/
     robots.txt). No agent journey through this domain is possible.

9. RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   IMMEDIATE ACTION REQUIRED
   ──────────────────────────────────────────────────────────────────

   • Investigate and correct the DNS configuration for
     advertising.roku.com. Confirm that the A/CNAME record points to
     Roku-owned infrastructure. If the domain was previously hosted on
     a shared CDN platform and that contract lapsed, the IP range is
     likely being reused by other tenants on the same CDN. Roku's DNS
     team should audit all advertising.roku.com DNS records immediately.

   • If advertising.roku.com is intentionally decommissioned, set up a
     301 permanent redirect to the current Roku advertising platform URL
     (e.g., ads.roku.com, oneview.roku.com, or a relevant landing page
     on roku.com). Do not allow the domain to serve third-party content.

   • Request removal from any shared CDN IP pool that is currently
     serving competitor content at this address.

   ONCE INFRASTRUCTURE IS RESTORED
   ──────────────────────────────────────────────────────────────────

   • Create /llms.txt covering Roku's advertising ecosystem: OneView DSP
     platform overview, streaming TV ad formats (display, video, overlay),
     audience segments, measurement and attribution capabilities, and
     developer/API documentation. Roku's CTV-first position makes this
     a high-value document for agents in media-buying workflows.

   • Add Organization JSON-LD to the homepage identifying Roku's
     advertising division, its relationship to parent Roku Inc., and
     sameAs references to LinkedIn, official press pages, and SEC filings.

   • Ensure robots.txt is Roku-authored and includes explicit Allow rules
     for GPTBot, ClaudeBot, and Google-Extended to signal openness to
     AI-mediated discovery.
