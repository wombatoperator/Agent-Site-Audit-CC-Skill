   Grade : F    Score : 2 / 100

   Quantcast.com, like the audited Lotame.com domain, is not a functioning
   standalone website. The domain performs an immediate JavaScript redirect
   to rotating third-party adtech company homepages. Observed redirect
   targets during this audit included DoubleVerify, StackAdapt, IAB Tech
   Lab, and Human Security (formerly White Ops). The initial browser title
   "Quantcast | DSP Platform | DSP Programmatic Advertising" briefly
   appears as the page loads before the redirect fires. No usable homepage
   content, accessibility tree, structured data, navigation, or interactive
   elements are present on the quantcast.com domain itself.

   This audit scores www.quantcast.com as the domain-under-test. All
   scoring dimensions reflect the state of that domain, not the rotating
   redirect targets. In contrast to Lotame.com (which returned an empty
   robots.txt), the robots.txt and llms.txt behavior for quantcast.com
   was more severe: the /robots.txt path was itself subject to redirect
   behavior and ERR_ABORTED errors on navigation, making even standard
   crawl-guidance discovery impossible for bots. This domain scores lower
   than Lotame.com on every dimension.

   SPECIAL NOTE: Quantcast was historically a significant audience
   measurement and programmatic advertising company. The current state of
   the www.quantcast.com domain — a rotating JavaScript redirect with no
   stable content — suggests the domain has been repurposed or is in an
   unmanaged post-acquisition or partnership state.

8. TOP 3 CRITICAL ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ① JavaScript redirect intercepts all paths including /robots.txt
     Unlike Lotame.com (where the robots.txt URL at least returned a
     stable empty response), Quantcast.com's redirect mechanism intercepts
     even /robots.txt requests and redirects them to third-party pages.
     This is a uniquely severe infrastructure failure: robots.txt is the
     foundational crawl-control mechanism relied upon by every search
     engine and AI crawler. Breaking it means zero crawl governance.
     The redirect also causes ERR_ABORTED on page.goto calls, making
     the domain unusable for headless automation of any kind.

   ② Complete content void — no Quantcast content accessible at any path
     The rotating redirect ensures no path under www.quantcast.com serves
     Quantcast-branded content. An AI agent researching "Quantcast real-time
     bidding" or "Quantcast audience measurement" would be sent to
     DoubleVerify, StackAdapt, or IAB Tech Lab with no indication of the
     redirect. This is worse than a 404 — at least a 404 tells the agent
     the resource doesn't exist. The redirect actively misdirects.

   ③ networkidle timeout — hardened agent failure mode
     The rotating redirect cycle (multiple "commit" + "domcontentloaded"
     + "load" event chains) causes page.waitForLoadState to exceed its
     25-second timeout. This is the most disruptive technical failure
     observed across this audit series. Not only does the domain have no
     content — it actively breaks the wait-for-stability mechanism that
     agents use to know when a page is ready to interact with.

9. RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   IMMEDIATE  (domain is currently broken for all automated agents)
   ──────────────────────────────────────────────────────────────────

   • Disable the JavaScript rotating redirect immediately
     Replace with either: (a) a 301 server-side redirect to a single,
     canonical destination, (b) a static informational page about the
     domain's current status, or (c) a 410 Gone if the domain is being
     decommissioned. Client-side rotating redirects are an anti-pattern
     that breaks robots.txt, sitemaps, and agent navigation.

   • Restore a functional robots.txt
     The /robots.txt URL must return a text/plain response, not an HTML
     redirect. A minimal valid robots.txt (even "User-agent: *\nDisallow:")
     is infinitely better than a rotating redirect serving StackAdapt's
     homepage to crawlers requesting crawl rules.

   • Fix networkidle compatibility
     The repeated navigation event cycles must be resolved. Any agent or
     automated tool that uses wait-for-networkidle (the industry standard
     for page-load detection) will time out on this domain. This breaks
     accessibility scanners, SEO crawlers, uptime monitors, and AI agents
     simultaneously.

   QUICK WINS  (once domain serves stable content)
   ──────────────────────────────────────────────────────────────────

   • Add Organization JSON-LD and OG meta tags
     A minimal homepage or landing page with Organization schema
     (name: "Quantcast", description, url, logo, sameAs) and four OG
     tags would immediately provide AI agents with actionable identity
     signals for the Quantcast brand.

   • Create /llms.txt
     Point to any surviving Quantcast product documentation or the
     canonical destination for Quantcast's audience measurement tools.
     Even a stub file signals AI readiness and is better than a 404.

   STRATEGIC  (brand preservation in AI-mediated research)
   ──────────────────────────────────────────────────────────────────

   • Audit all redirect targets for brand-alignment risk
     Quantcast's rotating redirect sends visitors to DoubleVerify, IAB
     Tech Lab, Human Security, and StackAdapt — all legitimate adtech
     companies, but none of them Quantcast. An AI agent answering "what
     does Quantcast do?" after following a Quantcast.com redirect to
     DoubleVerify would produce a materially incorrect answer. The brand
     damage from misdirected AI indexing compounds over time.

   • Consider a branded acquisition/transition landing page
     A single static HTML page at www.quantcast.com with the company's
     history, acquisition details, and links to where Quantcast's technology
     now lives would preserve brand equity in AI search and score
     significantly higher across all audit dimensions.
