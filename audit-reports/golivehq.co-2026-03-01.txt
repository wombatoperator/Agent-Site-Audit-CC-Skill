8. TOP 3 CRITICAL ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ① ZERO proactive AI readiness infrastructure
     Neither /agents.md nor /llms.txt is present. These 13 missing points
     are the largest single contributor to the C grade. Both files require
     no platform changes — they're plain-text, can be hosted externally and
     proxied, and signal to agent operators that GoLive is agent-friendly.
     In an ad-tech and agent-shopping context, this matters: agents will
     increasingly check for agents.md before attempting interactions.

   ② JSON-LD schemas are Squarespace stubs with empty critical fields
     Both WebSite.description and LocalBusiness.address/openingHours are
     empty strings. An AI agent (or LLM reading structured data) sees a
     business with no description and no location — untrustworthy signal.
     This is a content problem, not a code problem. The schema markup exists;
     it just needs to be populated in Squarespace's structured data settings.

   ③ Heading hierarchy inversion in the "How It Works" section
     A Squarespace template quirk causes the page to jump h2 → h4 and then
     nest h3 inside h4 (inverted). Agents using heading structure to build
     a page outline will produce a malformed outline. Screen readers and
     agent navigation systems that traverse headings sequentially will
     encounter this as noise. This requires a template/CSS-level fix.

9. RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   QUICK WINS  (implement in < 1 week, no platform rebuild required)
   ──────────────────────────────────────────────────────────────────

   • Create /agents.md at the site root
     Declare: permitted automated interactions (browsing, product lookup,
     booking flow), rate limits, preferred entry points, contact for agent
     operators. Host on a CDN or proxy from Squarespace's code injection
     if direct root file hosting is unavailable. Gains up to 8 pts.

   • Create /llms.txt at the site root
     List primary content sections: Template Shop, Custom Design, Portfolio,
     Blog, About — with URLs and 1-line descriptions. Follows the llms.txt
     spec (https://llmstxt.org). Gains up to 4 pts.

   • Populate the Squarespace LocalBusiness + WebSite schema fields
     In Squarespace → Settings → SEO → Structured Data, fill in:
     description (WebSite), address, and business hours (LocalBusiness).
     This costs 5 minutes and moves the JSON-LD score from 4/8 to ~7/8.

   • Add aria-label to image-only template gallery links (e25, e27, e29)
     Each unlabeled thumbnail link should carry aria-label="View [Template Name]
     template" matching its sibling text link. Can be done via Squarespace
     code injection targeting those specific anchor elements.

   STRATEGIC  (longer-term, higher engineering effort)
   ──────────────────────────────────────────────────────────────────

   • Fix the heading hierarchy in the "How It Works" section
     Change the section label from h4 to h2 (or remove as a heading and
     use a styled paragraph). The step headings (1, 2, 3) should be h3.
     This is a Squarespace template-level change — may require editing the
     underlying template or injecting CSS that demotes/promotes the elements.

   • Add aria-label to distinguish primary nav from footer nav
     The two <nav> elements are semantically identical to agents. Add:
     <nav aria-label="Primary navigation"> and
     <nav aria-label="Footer navigation">
     via Squarespace code injection wrapping or template override.

   • Add a semantic <footer> element (or role="contentinfo")
     Currently footer content lives in a generic container. Agents looking
     for footer landmarks will not find one. Add role="contentinfo" to the
     outermost footer container via code injection.

   • Resolve the "SHOP TEMPLATES" anchor-link ambiguity
     The sticky nav "SHOP TEMPLATES" link behaves as a scroll anchor rather
     than a route to /shop. Either: (a) make it navigate to /shop, or
     (b) add aria-label="Browse template shop" to distinguish intent from
     the name "SHOP TEMPLATES" which implies a distinct destination.
