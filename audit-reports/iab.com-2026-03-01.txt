1. EXECUTIVE SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   Grade : C    Score : 67 / 100

   IAB is the defining contradiction of this audit series. As the industry
   body that literally sets digital advertising standards — and has
   published extensively on AI's role in the future of media — they ship
   a comprehensive /llms.txt, a rich interconnected JSON-LD @graph, and
   a SearchAction schema. Then they actively block ClaudeBot, GPTBot, and
   every major AI crawler in their robots.txt. The site's operational
   agent-accessibility is also genuinely weak: the heading hierarchy is
   effectively random, dozens of interactive elements share identical or
   meaningless labels, and most research content sits behind an
   account gate that agents cannot pass. The score is pulled up by strong
   structured data and zero bot resistance; it is dragged down by an
   accessibility tree that would challenge any automated system.

   The tension here is significant and worth flagging to your client.

   the explicit AI bot blocks zero out the robots.txt sub-score.

8. TOP 3 CRITICAL ISSUES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   ① Heading hierarchy is non-functional for agents
     The page has no effective h1 in main content (the logo link h1 is
     empty text). Content headings jump: h3 → h5 → h4 → h3 → h2, in that
     order. This is heading levels used as CSS font classes. Any agent
     using document structure to understand the page — summarising,
     navigating by section, or building a content model — will fail.
     This is a WordPress theme-level issue requiring a template audit.

   ② "+N", "Menu", and "Close" buttons make card interactions
     non-deterministic for agents
     With 8 identical "Menu" links, 8 identical "+N" count buttons,
     and 5 identical "Close" buttons, an agent cannot reliably
     interact with a specific card. It can find all "Menu" buttons
     but has no way to know which card each belongs to without
     inspecting DOM ancestry — which defeats the purpose of the
     accessibility tree. This pattern is pervasive across the content
     grid and affects every research/news card on the site.

   ③ Virtually all research content is gated
     During deep navigation, every research report encountered the
     "Claim your free account" gate before substantive content was
     visible. An AI agent operating on behalf of a buyer, researcher,
     or journalist has no path to IAB research content without
     first authenticating. Only the Annual Reports page offers
     public PDFs. This is a business model decision, not purely a
     technical one, but it severely limits agent utility.

9. RECOMMENDATIONS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

   QUICK WINS  (implement in < 1 week)
   ──────────────────────────────────────────────────────────────────

   • Fix og:title — change "Home" to "IAB: Empowering the Media &
     Marketing Industries" in WordPress SEO settings. This is a
     one-field change in Yoast/RankMath and affects every social
     share from the homepage.

   • Add twitter:title and twitter:description meta tags to match
     the OG equivalents — two lines of code.

   • Add aria-label to the <nav> element: "Primary navigation".
     Single attribute, significant improvement for agents parsing
     multi-page navigation structure.

   • Add aria-label to region elements — at minimum, give the
     featured content carousels/regions a descriptive label so
     agents can distinguish sections.

   STRATEGIC  (requires theme/template-level work)
   ──────────────────────────────────────────────────────────────────

   • Fix the heading hierarchy — this is the highest-impact structural
     change. The page needs:
     - h1 in <main> (not the logo): should be the page's primary topic
       (e.g. "Empowering the Digital Advertising Industry")
     - h2 for major page sections (Upcoming Events, Featured Research,
       Getting Involved, Knowledge Center)
     - h3 for items within those sections
     Current h3/h4/h5 usage should be mapped to h2/h3 based on
     visual hierarchy. This is a WordPress template/block pattern change.

   • Replace "+N" count buttons with descriptive aria-labels
     Instead of "+3", use aria-label="Show 3 more tags for [article title]"
     via data attributes in the template. This requires a small template
     change but eliminates the most agent-hostile pattern on the site.

   • Disambiguate "Menu" links with aria-label tied to card title
     Each card's options/kebab menu should use aria-label="Options for
     [article title]". Template-level change, one line per card component.

   • Consider an agent-accessible content tier
     IAB's research is its primary value proposition. Making report
     abstracts, key findings (top 3 bullets), and methodology summaries
     publicly available without gating would massively increase agent
     utility. A structured summary section before the login gate —
     even 200 words — would allow agents to surface IAB insights in
     AI-mediated research and buying workflows.

   • Populate the Organization schema
     Add to the existing @graph: description ("The trade association
     for the digital advertising industry, founded 1996, 700+ members"),
     foundingDate: "1996", legalName: "Interactive Advertising Bureau",
